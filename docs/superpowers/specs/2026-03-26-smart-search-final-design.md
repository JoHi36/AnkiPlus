# Smart Search — Final Design Spec

**Date:** 2026-03-26
**Status:** Draft
**Feature:** Unified search with AI answer + semantic card network + dynamic stack creation

## Overview

The search bar on the Stapel tab becomes a "Google for your cards": type a question or term → get a concise AI answer from your own cards + a visual network of the 25 most relevant cards → start a dynamic stack with one tap.

## Core Principles

1. **One input, three outputs:** Answer + Network + Stack
2. **Honest:** "Diese Frage kann mit deinen Karten nicht beantwortet werden" if not enough context. No filler, no hedging.
3. **Fast first, polished second:** Graph appears immediately, AI answer arrives 1-2s later
4. **Invisible Addiction:** The search bar morph animation is the signature moment

## UX Flow

### State 1: Default (Heatmap)

```
┌──────────────────────────────────────────┐
│  Anki.plus                  [Deck-Liste] │
│  ┌────────────────────────────────────┐  │
│  │ 🔍 Was willst du lernen?       ⌘K │  │  ← Frosted input, centered

│                                          │
│  ┌────────────────────────────────────┐  │

│  │       (weakness areas)            │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

### State 2: Search submitted (Morph Animation)

1. User types "Cortisol" → presses Enter
2. **The search bar morphs:** slides down from its centered position to the bottom of the screen, becoming the standard input dock (same ChatInput component used everywhere)
3. **Simultaneously:** Heatmap fades out, card network materializes from center with staggered animation
4. **The morph is smooth** — the frosted glass element physically travels from top-center to bottom, reshaping as it goes (~500ms cubic-bezier)

```
┌──────────────────────────────────────────┐
│  Anki.plus                  [Deck-Liste] │
│                                          │
│           ○  ○    ○                      │
│          ○ [Cortisol] ○                  │  ← Query node in center
│           ○  ○  ○  ○                     │     Cards as colored nodes
│          ○    ○     ○                    │     Star topology
│                                          │
│  ┌────────────────────────────────────┐  │
│  │ ✕ │ Kurzantwort...      │ Stapel  │  │  ← Morphed input dock
│  │   │                     │ starten │  │
│  └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

### State 3: Answer arrives

The AI answer replaces the typed text in the input dock:

```
┌─────────────────────────────────────────────────────┐
│ ✕ │ Cortisol: Glucocorticoid der NNR,        │ 25 relevante │
│   │ katabol, immunsuppressiv, Stresshormon.   │ Karten kreuzen│
└─────────────────────────────────────────────────────┘
```

- **Left:** ✕ button → dismiss, return to heatmap
- **Center:** Concise answer (max 2-3 sentences)
- **Right:** "25 relevante Karten kreuzen" button

### State 4: Next query

- Press Enter or tap input area → answer clears, cursor appears, graph stays
- Type new query → new search, graph transitions to new results
- Press ✕ or Escape → everything dismisses, back to heatmap

## Input Dock Behavior

The input dock at the bottom uses the existing **ChatInput component** (`.ds-frosted` material). It has three states:

### Searching state
```
[🔍] [Cortisol________________________] [⏳]
```
Spinner on right while searching.

### Answer state
```
[✕] [Cortisol: Glucocorticoid der...] [25 Karten kreuzen]
```
Answer replaces input text. Action button becomes "Karten kreuzen".

### Ready for next query
```
[🔍] [_________________________________] [⏎]
```
Empty input, ready for next search. Previous graph stays visible.

## AI Answer Generation

### Prompt Strategy

The answer is generated by the existing Tutor agent via a **concise mode**:

**For single terms** (no question words detected):
```
Definiere '{term}' in maximal 2 Sätzen basierend auf diesen Lernkarten.
Wenn die Karten keine klare Definition liefern, antworte:
"Keine Definition in deinen Karten gefunden."
Kein Drumherum, keine Lückenfüller.
```

**For questions:**
```
Beantworte diese Frage in maximal 2-3 Sätzen basierend auf diesen Lernkarten:
"{query}"
Beantworte NUR den Kern der Frage. Wenn die Karten nicht genug Kontext bieten,
antworte: "Diese Frage kann mit deinen Karten nicht beantwortet werden."
Kein Drumherum, keine Vermutungen.
```

### Honesty Rules
- If < 3 relevant cards found: "Nicht genug Karten zu diesem Thema."
- If cards don't clearly answer the question: "Diese Frage kann mit deinen Karten nicht beantwortet werden."
- Never fabricate answers from general knowledge — only use card content
- Offer: "Internet-Suche starten?" as fallback (triggers Research Agent — future feature)

### Progressive Loading
1. **Immediately (<1s):** Card search results → graph appears with basic term-based cluster labels
2. **Parallel (~2-3s):** LLM generates concise answer + semantic cluster labels
3. **When LLM responds:** Answer fades into input dock, cluster labels update smoothly

## Card Network (3D Graph)

### Structure: Star topology with semantic clusters

**Query node:** White, centered, shows the search text
**Cluster nodes:** Medium-sized, labeled by LLM (or by most frequent term as fallback)
**Card nodes:** Small, colored by deck, connected to their cluster

```
                    ○ Card
                   ╱
        [Synthese] ── ○ Card
       ╱           ╲
      ╱              ○ Card
[Query] ─── [Wirkung] ── ○ Card
      ╲              ╱
       ╲            ○ Card
        [Erkrankung] ── ○ Card
                    ╲
                     ○ Card
```

### Cluster Computation (Backend)

1. Find 25 cards via hybrid search (vector + SQL)
2. Compute pairwise cosine similarity between all 25 cards
3. Build clusters: connected components at similarity > 0.45
4. Each cluster gets a temporary label (most frequent shared term)
5. Send to frontend: `{ clusters: [{ label, cards: [...] }], query }`
6. In parallel: LLM generates better cluster labels → pushed via ankiReceive update

### Cluster Interaction

- **Hover on cluster node:** Shows cluster label + card count
- **Click on cluster node:** Zooms in, shows only that cluster's cards + "X Karten kreuzen"
- **Click on card node:** Shows card preview (question text + deck name)

### Visual Design

- Query node: white (#FFFFFF), slightly glowing, largest
- Cluster nodes: semi-transparent, colored by dominant deck, medium size
- Card nodes: solid deck color, small
- Edges: thin, low opacity (0.10-0.15), white
- Animation: `warmupTicks(0)` for live physics, `zoomToFit()` after settling
- Auto-rotate at 0.5 speed
- Background: transparent (page background shows through)

## Search Bar Morph Animation

The signature UX moment. Technical approach:

1. The search bar starts as a centered element in the page flow (position: relative)
2. On submit: measure its current position (`getBoundingClientRect`)
3. Clone it to a fixed-position element at the exact same screen coordinates
4. Hide the original
5. Animate the clone to the bottom position (left: 0, bottom: 0, width: 100%)
6. When animation completes, replace clone with the real ChatInput component
7. Duration: 500ms, easing: cubic-bezier(0.4, 0, 0.15, 1)

The reverse animation (✕ pressed) plays backwards.

## Bridge Methods

### Existing (reused)
- `searchCards` — hybrid search, returns cards + edges (modified for cluster output)
- `sendMessage` — Tutor answer (modified for concise mode)

### Modified: `searchCards` response format
```json
{
  "type": "graph.searchCards",
  "data": {
    "clusters": [
      {
        "id": "cluster_0",
        "label": "Synthese",
        "cards": [
          { "id": "123", "question": "...", "deck": "Biochemie", "score": 0.85 }
        ]
      }
    ],
    "query": "Cortisol",
    "totalFound": 25
  }
}
```

### New: `quickAnswer` message
```python
'quickAnswer': self._msg_quick_answer,
```

Generates a concise 2-3 sentence answer using the search results as context:
```json
{
  "type": "graph.quickAnswer",
  "data": {
    "answer": "Cortisol ist ein Glucocorticoid...",
    "answerable": true,
    "clusterLabels": { "cluster_0": "Biosynthese", "cluster_1": "Wirkungen" }
  }
}
```

If `answerable` is false: `"answer": "Diese Frage kann mit deinen Karten nicht beantwortet werden."`

## Integration with Existing Features

### Stapel Tab
- Deck list stays as-is when search is inactive
- Heatmap replaces empty state (knowledge weakness visualization)
- Search transforms the view: deck list/heatmap → card network + input dock

### Session Tab
- Unaffected. Full Tutor chat lives here.

### Statistik Tab
- Knowledge Heatmap gets its permanent home here (copy from Stapel default)
- "Schwächen lernen" creates stacks from weak areas

## Phases

1. **Search morph animation** — input bar transforms from top to bottom
2. **Cluster computation** — backend groups cards into semantic clusters
3. **Quick answer** — concise LLM answer in the input dock
4. **Cluster labels** — LLM-generated labels replace term-based ones
5. **Polish** — animations, edge cases, error states
